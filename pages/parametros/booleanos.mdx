---
title: "Booleanos"
---

Os booleanos desempenham um papel crucial na escolha das funções API a serem acionadas. Atribuindo um valor booleano "true", o sistema é capaz de identificar o tipo específico de API que será chamada. Isso simplifica a interação com a API e permite que o sistema execute a ação desejada. Abaixo estão exemplos de booleanos utilizados para identificar funções específicas da API:

**Exemplos de Funções com Booleanos:**

* **editeStatus (Editar Status):** Atribuir `true` a este booleano indica a intenção de editar o status do usuário.

* **altereNome (Alterar Nome):** Atribuir `true` a este booleano sinaliza o desejo de alterar o nome de usuário.

* **obterImagem (Obter Imagem):** Quando este booleano é definido como `true`, a API compreende que a solicitação é para obter uma imagem.

* **obterStatus (Obter Status):** Atribuir `true` a este booleano solicita a recuperação do status do usuário.

* **desBlock (Desbloquear):** Este booleano, quando definido como `true`, indica que a intenção é bloquear alguém.

* **statusWebhook (Status do Webhook):** Atribuindo `true` a este booleano, o sistema reconhece que a solicitação é para ativar o webhook.

* **cancelarAgendamento (Cancelar Agendamento):** Quando este booleano é `true`, a solicitação é para cancelar um agendamento existente.

* **listaAgendamento (Listar Agendamento):** Atribuir `true` a este booleano solicita a listagem de agendamentos disponíveis.

* **grupo (Grupo):** Quando este booleano é definido como `true`, a solicitação envolve operações em grupos.

* **criarGrupo (Criar Grupo):** Este booleano indica a intenção de criar um novo grupo quando definido como `true`.

* **gerenciarGrupo (Gerenciar Grupo):** Atribuir `true` a este booleano informa o sistema de que a solicitação envolve a administração de grupos.

* **tituloGrupo (Título do Grupo):** Quando este booleano é definido como `true`, a solicitação está relacionada à modificação do título de um grupo.

* **descGrupo (Descrição do Grupo):** Este booleano, quando definido como `true`, indica que a solicitação é para atualizar a descrição de um grupo.

* **permissaoGrupo (Permissão do Grupo):** Atribuir `true` a este booleano sinaliza a intenção de gerenciar as permissões de um grupo.

* **sairGrupo (Sair do Grupo):** Quando este booleano é `true`, a solicitação é para sair de um grupo específico.

* **obterConvite (Obter Convite):** Este booleano indica que a intenção é obter um convite para um grupo quando definido como `true`.

* **entrarConvite (Entrar no Convite):** Atribuir `true` a este booleano solicita a ação de entrar em um grupo por meio de um convite.

* **infoConvite (Informações do Convite):** Quando este booleano é definido como `true`, a solicitação está relacionada à obtenção de informações sobre um convite de grupo.

* **dispositivos (Dispositivos):** Atribuir `true` a este booleano informa o sistema de que a solicitação envolve a interação com dispositivos.

* **desconectar (Desconectar):** Quando este booleano é definido como `true`, a solicitação é para desconectar um dispositivo específico.

* **editeDispositivo (Editar Dispositivo):** Este booleano indica a intenção de editar as configurações de um dispositivo quando definido como `true`.

* **apagarDispositivo (Apagar Dispositivo):** Atribuir `true` a este booleano solicita a exclusão de um dispositivo da lista.

Esses booleanos são ferramentas essenciais para direcionar e acionar funções específicas da API, simplificando a interação com o sistema e permitindo a execução de ações desejadas de forma eficaz.
